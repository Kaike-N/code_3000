#Assessment Risk
    - This github repo does not store sensitive personal data, authentication credientials, or any confidential information. It only contains coursework assignments and related code. If the information in this repo fell into the wrong hands, there could be plagiariasm or academic integrity violations if others submit similar work. However, disregarding these minor issues the overall security risk is low.

#Steps to Secure Repo
    - No sensitive information has been stored in this repo. 
    - Changes are tracked through version control
    - The repository includes a CODEOWNERS file inherited from the original fork, which identifies the professor as a code owner.
    Because the repo is for academic purposes and does not store sensitive data, advanced security controls (like strict pull request rulesets) are not necessary.
